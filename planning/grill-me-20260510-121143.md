# Grill Session: ApexTuts v1

Date: 2026-05-10
Source: idea.md

## Decisions Captured
1. Primary v1 success metric: visitor-to-registration conversion.
2. Target: 8% conversion within 60 days.
3. Primary segment: non-technical working adults (25–45) upskilling for work.
4. Primary acquisition: organic social + newsletter funnel.
5. Registration friction: email + first name only.
6. Progress identity model: localStorage only.
7. Tradeoff accepted: single-device progress, no cross-device continuity.
8. Failure policy: explicit warning that progress is non-recoverable if data is cleared/switched.
9. Enrollment model: free enrollment with immediate access.
10. Course structure: class -> modules -> lessons.
11. Completion event: explicit "Mark as complete".
12. Navigation: free navigation with recommended sequence.
13. Lesson content types: text + embedded video + downloadable resources.
14. Discovery: search + topic filters + difficulty tags.
15. Required class metadata: slug, title, short description, instructor, duration, difficulty, topic tags, thumbnail, published flag.
16. Draft policy: published=true only; drafts hidden.
17. Content operations: seeded SQLite data + manual admin scripts.
18. Routing: React Router with nested routes.
19. Zustand boundary: minimal global store (enrollment/progress/session context + UI prefs only).
20. Analytics events: full funnel + lesson started/completed.
21. Analytics storage: SQLite event table in app DB.
22. Privacy posture: minimal analytics + plain-language privacy notice.
23. Accessibility: deferred (explicitly put aside for now).
24. Design guardrails: neutral base + one accent, max 2 font families, strict spacing scale, no animated backgrounds.
25. Color direction: cool neutrals + teal accent.
26. Empty catalog handling choice: seed demo classes.
27. Demo class trust mitigation: label as "Preview Class" + waitlist CTA (no real enrollment).
28. Scope boundary: strict v1 includes marketing, catalog, class detail, lesson player, progress tracking, enrollment. Excludes auth, payments, quizzes, admin UI.
29. Deployment target: local/private only for now.
30. Readiness gate: end-to-end demo pass (discover -> enroll -> start -> complete -> persists after refresh).
31. Content ownership: one owner + weekly update checklist.
32. Post-v1 progress migration posture: wait for complaints.
33. Hard trigger for migration: none.
34. Permanent localStorage limitation accepted.
35. Required user warning text accepted: 
   "Progress is saved only on this browser and device in v1. Clearing browser data or switching devices will reset progress."

## High-Risk Choices (Explicitly Accepted)
- Permanent localStorage-only progress with no migration trigger.
- Accessibility baseline deferred.
- Local/private deployment only (no production readiness target yet).

## Immediate Implementation Implications
- Must implement prominent progress limitation warnings at first lesson start and FAQ.
- Analytics and progress metrics may be skewed by device/browser resets.
- Content publishing requires disciplined manual scripts/checklist due to no admin UI.
